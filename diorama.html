Diorama Page

Each object now has an explicit page position.

The wrapper decides where an object lives on each page. The child still uses the same grow and spin values, so page transitions are easier to tune without losing the simple class-and-data-attribute API. 

<div class="scene-node effect-slide" data-page-1-x="14%" data-page-1-y="28%">
  <div class="diorama-sun effect-target effect-grow" data-page-1-scale="0.92"></div>
</div>

Positioned Page States

The same objects move to explicit coordinates on every page.

0%, 0%
100%, 0%
50%, 50%
0%, 100%
100%, 100%

Page positions now live directly on the wrapper: data-page-1-x, data-page-1-y, data-page-2-x, and so on.

The child element still uses the existing effect classes for visual changes like scale, while the wrapper slides to the next coordinate.
